# 長峰保育園ホームページ制作（WordPressテーマ開発 / Docker環境構築）

## 概要

長峰保育園の既存サイトが静的なHTMLで構築されており、
職員による情報更新が困難であるという課題を解決するため、
WordPressを導入しリニューアルを想定して制作しました。

保護者の年齢を20代〜30代と想定し、PC版だけでなく、スマホ版も実装しました。
お知らせとBLOGの一般公開情報と、「行事写真」や「園だより」といった保護者向けに分けた設計をしています。
Docker環境で構築しており、クローン後すぐに動作確認が可能です。

## 開発経緯

既存の長峰保育園のサイトは、当時の園長の知人に依頼して作成された静的なHTMLサイトでした。

そのため、保育園の職員自身で更新することができず、
情報が長期間更新されていない状態となっており、
保護者にとって有益な情報が十分に提供できていないという課題がありました。

そこで、WordPressを導入し、
職員が管理画面から簡単に情報更新できる仕組みを構築することを目的として本サイトを制作しました。

また、実在する保育園（義母が園長を務める長峰保育園）を題材にすることで、
実務を想定した設計・実装を行いました。

## 主な機能

### 一般公開機能
 **お知らせ(news)**

  →今までサイト上から園のお知らせができなかったので、職員が管理画面から簡単に情報発信できる仕組みとして実装しました。

  →本機能では、管理画面のお知らせから、タイトルと本文を簡単に入力投稿できる仕組みを実装し、園からの情報発信を効率化しました。

  →今後は在園者だけでなく、一般の方にも情報や行事の発信もできるようになります。

 **お問い合わせ**

  →今まで一般の方の保育園へのお問い合わせが電話だけでしかできなかったので、サイト上から問い合わせできるように実装しました。

  →問い合わせ種類にラジオボタンで主な問い合わせ内容を実装することで一目で内容を確認できるようにしています。

  →電話が苦手な方や、電話できない状況でも問い合わせがしやすくなります。

  誰でも入力できるようにし、未入力の場合バリデーションが出るようにしてます

  [▶ 動画を見る](https://github.com/user-attachments/assets/bd5c0d89-99da-4072-bd92-3900c1bc94e6)



 **ブログ**

  →今までは静的なサイトであったため、入園希望者や地域の方への園内の様子を発信する機能がありませんでした。

  →WordPressの通常投稿を実装することで、園の日常の雰囲気を発信することができます。

  →入園希望者の情報収集源にもなり、また地域の方との距離を縮めるきっかけにもなります。

### 在園者向け機能
 **園便り(letter)**

  →今まで紙で配布していましたお便りは園が印刷して保護者に渡していました。

  →紙で配布していたお便りをPDF化し、スマートフォンでの閲覧と印刷の両方に対応しました。（紛失防止にも対応）

  →園で印刷する手間も省け、過去のお便りも確認することもできます。

 **行事写真**

  →今まで園が行事で撮影していた写真を在園者に渡す方法がオフラインでしかできず、手間がかかっていました。

  →Envira Galleryはその行事の写真をサイト上で一覧表示して写真を確認し、必要に応じて保存することができます。

  →これにより、園児の写真を園側がプリントアウトして渡す手間が省け、保護者もスマホやPCですぐに手に入れることができます。

 **年間予定表**

  →今まで年間予定表をサイト上で閲覧できず、修正があっても保護者任せであったり、園から再配布の形をとらなければなりませんでした。

  →今回PDFファイルで園側はいつでも修正や追加して発信することができます

  →保護者が必要な情報をいつでも確認できるだけでなく、修正時の再配布の手間を削減し、園側の運用負担も軽減しています。

 **欠席・遅刻連絡**

  →欠席・遅刻連絡につきまして、現状は既存の専用アプリの方が管理や通知機能に優れているため、すぐに置き換えることは想定していません。ただし、将来的にシステムを一本化する可能性も考慮し、サイト側でも連絡機能を実装しました。まずは保護者の導線として活用し、段階的に運用を見ながら統合の検討ができる構成としています。

  →機能としましてコンタクトフォームと同様の仕組みを利用しつつ、利用目的を「欠席・遅刻連絡」に特化させることで、入力内容を明確にし、保護者・園双方の負担軽減を図っています。

### 管理者側機能

 **書式のダウンロード**

  → 今まで書式のダウンロードの表示がなく、保護者から問い合わせがあった時にお渡しするという手間が発生していました。

  → 一般公開のサイトから書式のダウンロードページに遷移して、PDFファイルをダウンロードできるようにしました。

  → 保護者も園に問い合わせすることなく、必要な書類を自分でダウンロード・印刷できるようになりました。

 **権限設計**

  本システムでは、誤操作や情報漏洩を防ぐために、ユーザーの役割ごとに操作権限を明確に分離しています。

  WordPressのユーザーロールをベースに、Membersプラグインを用いて権限のカスタマイズを行いました。

  - 園長・管理者

    → すべての操作が可能（投稿・編集・公開・ユーザー管理）

  - 主任

    → 投稿および公開が可能（コンテンツ管理を担当）

  - 先生

    → 投稿のみ可能（下書き作成まで）

  - 保護者

    → 在園者向けページの閲覧のみ可能

  また、「在園者向けページ」はログインユーザーのみに制限することで、外部からのアクセスを防いでいます。

  このように権限を段階的に制御することで、

  ・誤った公開や編集の防止  
  ・情報の公開範囲の適切な制御  
  ・非エンジニアでも安全に運用可能  

  な構成としています。

 **SEO対策**

  → Yoast SEOを使用し、各ページごとにタイトル・メタディスクリプションを設定

  → トップページおよびお知らせページでは、地域名（新潟県長岡市）や施設名を含めたキーワード設計を行い、検索エンジンからの流入を意識

  → 在園者向けページ（園だより・行事写真・年間予定・欠席連絡など）は、
  外部に公開する必要がない情報のため、noindex設定を行い検索結果に表示されないように制御

  → これにより、
  ・一般公開すべき情報は検索流入を狙う  
  ・内部向け情報は検索対象から除外する  
  という情報設計を行っています

## 環境構築

※本リポジトリはポートフォリオ用のため、ローカル環境での動作となります。

### 起動手順

```bash
git clone https://github.com/yuichihomma/nagamine-hoikuen-site.git
cd nagamine-hoikuen-site
docker compose up -d --build

```

本リポジトリには表示確認用データを含んだDBダンプを同梱しています。
コンテナ起動後、以下のコマンドを実行してください。
## DBインポート手順

まず MySQL コンテナ名を確認してください。

```bash
docker ps

```
その後、以下を実行してください。

```bash
cat sql/wordpress.sql | docker exec -i <mysqlコンテナ名> mysql -u root -prootpass wordpress

```

※ <mysqlコンテナ名> は docker ps で表示される MySQL コンテナ名に置き換えてください。

ブラウザで以下にアクセスしてください。
http://localhost:8080

※初回起動時はコンテナのビルドに時間がかかる場合があります。
※WordPressの初期設定画面が表示される場合があります。
